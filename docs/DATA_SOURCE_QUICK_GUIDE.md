# 数据源设置 - 快速说明

## 一句话总结

**数据源设置只影响板块列表，板块热度固定使用东方财富。**

## 功能对应关系

```
┌─────────────────────────────────────────────────────┐
│                   数据源设置                          │
│              (同花顺 / 东方财富)                      │
└─────────────────────────────────────────────────────┘
                        │
        ┌───────────────┴───────────────┐
        │                               │
        ▼                               ▼
┌──────────────┐              ┌──────────────┐
│  板块列表    │              │  板块热度    │
│ /api/sectors │              │/api/sectors/ │
│              │              │     hot      │
│  ✅ 受设置   │              │  ❌ 不受设置 │
│     影响     │              │  (固定东财)  │
└──────────────┘              └──────────────┘
```

## 为什么板块热度不受设置影响？

**原因**：板块热度需要以下数据：
- 实时涨跌幅
- 上涨家数
- 下跌家数
- 情绪得分计算

**现状**：
- ✅ 东方财富：提供所有数据
- ❌ 同花顺：不提供这些数据

**结论**：板块热度只能使用东方财富

## 你会看到什么

### 设置为同花顺时

**板块列表页面**：
```
✅ 同花顺: 获取到 90 个行业板块
```

**板块热度页面**：
```
⚠️ 东方财富板块行情失败: Connection aborted...
❌ 暂无数据，请检查网络连接或稍后重试
```

这是**正常现象**！因为：
1. 板块列表使用同花顺 ✅
2. 板块热度尝试东方财富 ❌（非交易时间失败）

### 设置为东方财富时

**板块列表页面**（交易时间）：
```
✅ 东方财富: 获取到 100 个行业板块
```

**板块热度页面**（交易时间）：
```
✅ 东方财富: 获取到 100 个板块行情
```

## 最佳实践

### 非交易时间
- **设置**：同花顺
- **可用功能**：板块列表、板块成分股
- **不可用**：板块热度（除非有缓存）

### 交易时间
- **设置**：东方财富或同花顺都可以
- **可用功能**：所有功能
- **建议**：运行 `update_board_cache.py` 更新缓存

## 常见问题

**Q: 为什么设置了同花顺还显示"东方财富失败"？**
A: 因为板块热度功能固定使用东方财富，这是正常的。

**Q: 如何在非交易时间查看板块热度？**
A: 使用缓存数据。在交易时间运行 `update_board_cache.py` 更新缓存。

**Q: 数据源设置有什么用？**
A: 影响板块列表和成分股的获取，同花顺在非交易时间更稳定。

**Q: 能让板块热度也使用同花顺吗？**
A: 不能，因为同花顺不提供实时涨跌幅和上涨下跌家数。
