# 数据源设置功能说明

## 重要说明 ⚠️

**板块热度功能**需要实时涨跌幅和上涨下跌家数，这些数据**仅东方财富提供**。

- **数据源设置**：仅影响板块列表和板块成分股的获取
- **板块热度**：无论选择哪个数据源，都会尝试从东方财富获取实时行情

## 功能概述

系统支持两个板块数据源：**同花顺**和**东方财富**，用户可以在设置页面自由切换。

## 数据源对比

| 特性 | 同花顺（推荐） | 东方财富 |
|------|--------------|---------|
| **板块列表** | | |
| 连接稳定性 | ⭐⭐⭐⭐⭐ 非常稳定 | ⭐⭐⭐ 交易时间稳定 |
| 非交易时间 | ✅ 可用 | ❌ 经常失败 |
| 板块数量 | 90个行业板块 | ~100个行业板块 |
| **板块热度** | | |
| 实时涨跌幅 | ❌ 不提供 | ✅ 提供 |
| 上涨下跌家数 | ❌ 不提供 | ✅ 提供 |
| 情绪得分 | ❌ 不提供 | ✅ 提供 |

## 使用建议

### 场景1：非交易时间查看板块列表 ✅
**推荐**：同花顺
- 连接稳定，随时可用
- 可以查看所有板块名称和代码
- 可以查看板块成分股

### 场景2：交易时间查看板块热度 ✅
**必须**：东方财富（自动使用）
- 提供完整的涨跌幅数据
- 包含上涨下跌家数
- 可计算板块情绪得分

### 场景3：非交易时间查看板块热度 ⚠️
**依赖**：缓存数据
- 板块热度需要东方财富数据
- 非交易时间API不可用
- 使用之前缓存的数据

## 设置方法

### 方法1：通过Web界面（推荐）

1. 访问系统设置页面
2. 在"数据源设置"区域选择：
   - **同花顺（推荐）** - 默认选项
   - **东方财富** - 需要交易时间
3. 点击"保存设置"

### 方法2：通过API

```bash
# 设置为同花顺
curl -X POST http://localhost:5001/api/settings/data-source \
  -H "Content-Type: application/json" \
  -d '{"source": "ths"}'

# 设置为东方财富
curl -X POST http://localhost:5001/api/settings/data-source \
  -H "Content-Type: application/json" \
  -d '{"source": "em"}'

# 查询当前设置
curl http://localhost:5001/api/settings/data-source
```

### 方法3：直接修改配置文件

编辑 `.cache/settings.json`：

```json
{
  "data_source": "ths"
}
```

可选值：
- `"ths"` - 同花顺
- `"em"` - 东方财富

## 自动降级机制

系统实现了智能降级：

1. **首选数据源失败** → 自动切换到备用数据源
2. **所有数据源失败** → 使用本地缓存
3. **缓存也不可用** → 返回明确错误提示

### 示例

用户设置：东方财富（首选）

```
尝试东方财富 → 失败（非交易时间）
  ↓
自动切换到同花顺 → 成功
  ↓
返回板块列表
```

## 默认设置

- **默认数据源**：同花顺（`ths`）
- **原因**：连接稳定，非交易时间也可用

## 注意事项

1. **数据源设置的作用范围**：
   - ✅ 影响：板块列表获取（`/api/sectors`）
   - ✅ 影响：板块成分股获取
   - ❌ 不影响：板块热度（`/api/sectors/hot`）- 固定使用东方财富

2. **板块热度功能**：
   - 需要实时涨跌幅和上涨下跌家数
   - 这些数据仅东方财富提供
   - 同花顺不提供这些数据
   - 无论选择哪个数据源，板块热度都会尝试东方财富

3. **为什么会看到"东方财富失败"**：
   - 即使设置了同花顺
   - 板块热度功能仍会尝试东方财富
   - 这是正常行为，不是bug

4. **缓存机制**：设置不影响缓存，缓存数据来自历史成功的API调用

5. **交易时间**：建议在交易时间使用东方财富更新缓存，非交易时间使用同花顺查看板块列表

## 常见问题

**Q: 为什么默认是同花顺？**
A: 同花顺连接更稳定，非交易时间也能正常使用。

**Q: 切换数据源后需要重启吗？**
A: 不需要，设置立即生效。

**Q: 如何获取完整的板块行情数据？**
A: 在交易时间切换到东方财富，或使用缓存的历史数据。

**Q: 两个数据源的板块列表一样吗？**
A: 基本一致，但可能有细微差异。东方财富约100个，同花顺90个。
