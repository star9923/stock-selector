# 网络错误处理指南

## 常见错误

### ConnectionResetError(10054)
**错误信息**: `远程主机强迫关闭了一个现有的连接`

**原因**:
- 网络不稳定
- API 服务器限流或拒绝连接
- 防火墙或代理干扰
- 请求超时

## 已实施的解决方案

### 1. 自动重试机制
所有网络请求已添加自动重试（最多3次），包括：
- 股票行情获取
- 板块数据获取
- 财务数据获取
- AI 分析请求
- 新闻数据获取

### 2. 超时保护
- 默认超时: 30秒
- AI API 超时: 60秒
- 自动恢复原始超时设置

### 3. 缓存降级
- 板块列表缓存: 30天
- 板块行情缓存: 24小时
- 失败时自动使用缓存数据

## 用户操作建议

### 遇到连接错误时
1. **等待重试**: 程序会自动重试3次
2. **检查网络**: 确保网络连接正常
3. **关闭代理**: 程序已自动禁用代理，但请确认系统代理已关闭
4. **稍后再试**: API 可能临时限流，等待几分钟后重试

### 减少错误发生
1. 使用稳定的网络环境
2. 避免高峰时段（交易时间）
3. 不要频繁运行程序
4. 使用缓存数据（程序自动处理）

## 仍然失败？

如果重试3次后仍然失败：
1. 检查网络连接
2. 尝试使用 VPN 或更换网络
3. 查看 `.cache` 目录是否有缓存数据
4. 联系开发者报告问题
