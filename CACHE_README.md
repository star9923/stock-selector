# 数据缓存说明

## 概述

系统实现了两层缓存机制：**实时API → 本地缓存**。不使用示例数据，确保所有数据都是真实的。

## 缓存类型

### 1. 板块情绪数据（热门板块排行）

**实时API（优先）**
- 直接从东方财富API获取最新数据
- 自动重试3次，支持指数退避
- 成功后自动保存到缓存

**短期缓存（6小时）**
- 位置：`.cache/get_board_sentiment.json`
- 有效期：6小时
- 当API失败时自动使用

**长期备用缓存（7天）**
- 位置：`.cache/board_sentiment_fallback.json`
- 有效期：7天
- 手动更新或API成功时保存

### 2. 板块成分股数据（板块详情）

**实时API（优先）**
- 从东方财富API获取板块成分股
- 自动重试3次

**板块缓存（7天）**
- 位置：`.cache/sector_stocks/{板块名称}.json`
- 有效期：7天
- 每个板块独立缓存

### 3. 个股历史数据（个股分析）

**实时API（优先）**
- 从东方财富API获取股票历史数据
- 自动重试

**个股缓存（1天）**
- 位置：`.cache/stock_history/{股票代码}.json`
- 有效期：1天
- 每只股票独立缓存

## 使用方法

### 自动模式（推荐）
系统会自动按优先级尝试各层缓存，无需手动干预。

### 手动更新缓存
当API可用时（交易时间），运行以下命令更新本地缓存：

```bash
python3 update_board_cache.py
```

脚本会：
1. 更新板块情绪数据
2. 询问是否更新热门板块成分股数据
3. 自动更新前10个热门板块的成分股

## 建议

1. **交易时间更新**：在交易日的交易时间（9:30-15:00）运行 `update_board_cache.py` 获取真实数据
2. **定期更新**：建议每周至少更新一次缓存
3. **网络问题**：如果API持续失败，检查网络连接和代理设置
4. **演示模式**：示例数据仅供演示，不应用于实际投资决策

## 故障排查

### 问题：一直显示"演示模式"
**原因**：所有缓存都已过期或不存在
**解决**：
1. 检查网络连接
2. 在交易时间运行 `update_board_cache.py`
3. 检查 `.cache/` 目录权限

### 问题：数据不更新
**原因**：使用了旧缓存
**解决**：
1. 删除 `.cache/` 目录下的缓存文件
2. 重新运行 `update_board_cache.py`

### 问题：API连接超时
**原因**：网络问题或API服务不可用
**解决**：
1. 检查是否在交易时间
2. 检查网络连接
3. 等待几分钟后重试
